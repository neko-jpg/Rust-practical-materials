# Level 6: 可観測性 (Observability)

## 目的
「動いているかどうか」だけでなく、「**どのように動いているか**」を本番環境で把握するための技術を学びます。
`println!` デバッグからの卒業を目指します。

## 学習項目

1. **Structured Logging (`tracing`)**:
    - 文字列ではなく、キーバリュー形式でログを出力する（JSON形式など）。
    - ログレベル (`debug`, `info`, `warn`, `error`) の適切な使い分け。
    - `Span` を使って、リクエストごとの処理の流れ（コンテキスト）を追跡する。

2. **Metrics**:
    - リクエスト数、処理時間、エラー数などを数値として計測する。

## 課題

`src/main.rs` を実行し、以下の挙動を確認してください。
1. JSON形式でログが出力されること。
2. 関数呼び出しの階層構造が `span` として記録されていること。
3. 非同期タスク内のログも適切に紐付けられていること。

## 実践
環境変数でログレベルを制御してみましょう。
```bash
RUST_LOG=debug cargo run
RUST_LOG=info cargo run
```
