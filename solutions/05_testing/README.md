# Level 5: テスト戦略 & CI

## 目的
「ただテストを書く」だけでなく、**保守性が高く、バグ発見能力の高いテスト** を設計する技術を学びます。

## 学習項目

1. **Unit Test**: 境界値分析と同値分割。
2. **Mocking**: `mockall` を使った依存関係の切り離し（外部APIやDBのモック化）。
3. **Property-based Testing**: `proptest` を使い、自動生成された大量の入力でエッジケースを発見する。
4. **CI (GitHub Actions)**: プルリクエスト時に自動でテストを実行するワークフローの作成。

## 課題 (`src/lib.rs`)

1. **`UserRegistry`**: 外部データベース（トレイトとして定義）に依存するユーザー登録ロジックを実装し、DBをモックしてテストしてください。
2. **`parse_email`**: メールアドレスのパース関数を実装し、`proptest` を使って「有効な形式なら常に成功し、無効なら失敗する」ことを検証してください。

## 実行
```bash
cargo test
```
