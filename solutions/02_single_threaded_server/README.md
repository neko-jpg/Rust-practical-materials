# Level 2: 低レイヤー通信と抽象化

このプロジェクトでは、標準ライブラリ (`std::net`) のみを使用して HTTP サーバーをゼロから実装します。
最初はシングルスレッドで動作させ、徐々にマルチスレッド化し、設計を抽象化していきます。

## 学習カリキュラム

### 2.1: 同期TCPサーバー
- `TcpListener` を使用してポートをバインドし、接続を受け付ける。
- `TcpStream` からの読み書き。

### 2.2: HTTPプロトコルの手動解析
- HTTP リクエスト（バイト列）を文字列として解析。
- ステータスライン、ヘッダー、ボディを含む HTTP レスポンスの手動構築。

### 2.5: トレイトによる抽象化
- **Goal**: リクエスト処理のロジックをハードコードせず、`Handler` トレイトとして定義する。
- **Key Concepts**: `trait`, `impl`, ジェネリクス。

### 2.8: マルチスレッド同期
- **Goal**: `ThreadPool` を実装し、複数のリクエストを同時に処理できるようにする。
- **Key Concepts**:
    - `std::thread`: スレッドの生成。
    - `std::sync::mpsc`: スレッド間通信（チャネル）。
    - `std::sync::{Arc, Mutex}`: データの安全な共有。
    - `Drop` トレイト: スレッドプールの安全なシャットダウン。
