# Level 3: 非同期プログラミング (Tokio)

Rust の非同期エコシステムの中心である `Tokio` を使って、リアルタイムチャットアプリケーションを構築します。
同期処理（スレッド）では難しかった「多数の同時接続」や「複雑なイベント待ち受け」を、効率的に記述する方法を学びます。

## 学習カリキュラム

### 3.1: 非同期ランタイムの基礎
- **Goal**: `async/await` 構文を理解し、`tokio::spawn` でタスクを立ち上げる。
- **Key Concepts**: Future, Task, Runtime, `.await`。

### 3.5: 高度な非同期制御
- **Goal**: `tokio::select!` を使用して、ネットワーク受信とユーザー入力を同時に処理する。
- **Key Concepts**: Concurrency vs Parallelism, `broadcast` チャネル。

## プロジェクト構成

- `src/bin/server.rs`: チャットサーバー。接続を受け付け、メッセージを全クライアントに配信します。
- `src/bin/client.rs`: チャットクライアント。サーバーと送受信しつつ、ユーザー入力も受け付けます。

## 実行方法

このプロジェクトには2つのバイナリが含まれています。

1. **サーバーの起動**:
   ```bash
   cargo run --bin server
   ```

2. **クライアントの起動** (別のターミナルで複数起動):
   ```bash
   cargo run --bin client
   ```
